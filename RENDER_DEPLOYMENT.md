# 🚀 คู่มือการนำระบบ Shopee Shorts Auto Poster ไปรันบน Render.com (100% Automate)

ไฟล์นี้จะช่วยแนะนำวิธีตั้งค่าระบบทั้งหมดเพื่อรันระบบบอทอัตโนมัติบน **Render.com** (ฟรีหรือโปรก็ได้) ได้อย่างเสถียรและไร้กังวลเรื่องโทเค็นหลุด!

---

## 📋 ข้อมูลสถิติต่างๆ สำหรับกรอกบนหน้าเว็บ Render.com

เมื่อเปิดสร้างโปรเจกต์ใหม่บน **Render** ให้เลือกสร้างเป็น **Web Service** จากนั้นกรอกข้อมูลดังต่อไปนี้:

| หัวข้อคอนฟิกบน Render | ค่าที่ต้องกรอก / เลือก | คำอธิบายเพิ่มเติม |
| :--- | :--- | :--- |
| **Runtime** | `Node` | รันด้วย Node.js ล่าสุด |
| **Build Command** | `npm install && npm run build` | ลงโปรแกรมเสริมและรวบรวมโค้ด TypeScript |
| **Start Command** | `npm start` | รันเซิร์ฟเวอร์ด้วยไฟล์ CJS ที่ถูกบีบอัดเรียบร้อย |

---

## 🔐 การกำหนดค่า Environment Variables (ตัวแปรสภาพแวดล้อม)

ให้ไปที่แถบ **Environment** ในหน้าควบคุมของ Render Web Service จากนั้นเพิ่มตัวแปร (Key-Value) ต่อไปนี้ เพื่อความปลอดภัยและรันระบบอย่างแท้จริง:

1. **`NODE_ENV`**: `production` (บังคับ เพื่อให้ระบบทำงานในโหมดเสิร์ฟไฟล์จริงและเสถียร)
2. **`GEMINI_API_KEY`**: *(คีย์ของ Gemini AI)* ดึงมาสุ่มประดิษฐ์ข้อความและแคปชันตลกสายฮากวนๆ
3. **`SHOPEE_AFFILIATE_ID`**: `15324930878` (หรือ Affiliate ID ของคุณเอง)
4. **`YOUTUBE_CLIENT_ID`**: *(คีย์ Client ID ของ Google Cloud)*
5. **`YOUTUBE_CLIENT_SECRET`**: *(คีย์ Client Secret ของ Google Cloud)*
6. **`YOUTUBE_REFRESH_TOKEN`**: *(Refresh Token สิทธิ์ถาวรที่เชื่อมโยงกับบัญชี YouTube Shorts ของคุณ)*

## 📌 ข้อแนะนำเพิ่มเติมเกี่ยวกับการจัดเก็บข้อมูล (Data Persistence)

* **Ephemeral Disk (แผ่นดิสก์ชั่วคราว):** เนื่องจากบน Render.com ในบริการแบบฟรีไดรฟ์จะถูกล้างค่าทุกครั้งที่มีการอัปเดตโค้ดหรือรีสตาร์ทเซิร์ฟเวอร์ (Ephemeral Disk) ดังนั้น ข้อมูลสินค้าใหม่ๆ ที่คุณกดเพิ่มผ่านหน้าจอ หรือ Logs การโพสต์ที่พึ่งบันทึก อาจจะคืนค่ากลับไปเป็นชุดเริ่มต้นตามโค้ดต้นฉบับได้
* **ทางแก้หากต้องการเก็บถาวร 100% บน Render:**
  1. **Render Disk (แนะนำสำหรับสิทธิ์เสียเงิน):** สามารถทำการเชื่อมต่อ **Persistent Disk** ของ Render เข้าที่โฟลเดอร์ `/src/db` ของคุณ ข้อมูลจะอยู่ครบตลอดชีพไม่หายไปไหนแน่นอน!
  2. **เชื่อมต่อฐานข้อมูลภายนอก:** หากต้องการรันฟรี 100% และข้อมูลไม่หาย สามารถปรับปรุงเพิ่มเติมให้เชื่อมต่อเข้ากับฐานข้อมูล Cloud อย่าง **Firestore (Firebase)** หรือ **MongoDB/Supabase** ได้อย่างง่ายดาย

## ⏰ ระบบตั้งเวลา (Cron Job) หลังบ้านรันอย่างไร?

ในระบบที่เราวางโครงสร้างไว้ให้ จะมีโมดูล `node-cron` ทำการดักจับเวลาในระดับเซิร์ฟเวอร์ทำงานตลอดเวลาดังนี้:

```typescript
// ทำงานทุกวันตอนเวลา 09:00 น. ตรง (เช้า)
cron.schedule("0 9 * * *", async () => {
  console.log("⏰ [Cron Job] 09:00 AM Triggered! Initiating YouTube Shorts post workflow.");
  await executeDailyAffiliateShortsPost();
});
```

* **โน้ตสำคัญเกี่ยวกับ Render Free Tier:** เซิร์ฟเวอร์ Render Free Tier จะมีการ **Sleep (หลับ)** หากไม่มีการเข้าเยี่ยมชมหน้าเว็บนานเกิน 15-30 นาที ซึ่งอาจทำให้สคริปต์ Cron Job ไม่ตื่นขึ้นมารันตอน 09:00 น.
* **วิธีแก้:**
  1. ใช้เว็บภายนอกช่วยปลุกฟรี (เช่น [UptimeRobot](https://uptimerobot.com/)) โดยให้ทำการยิง API ไปที่ลิงก์หน้าแรกของ Render ทุกๆ 5-10 นาที เพื่อให้เซิร์ฟเวอร์ออนไลน์ตลอดเวลาและสั่งงาน Cron Job ได้แม่นยำ
  2. หรือปรับปรุงเปลี่ยนไปสร้าง **Render Cron Job** (ระบบตัวจัดการตั้งเวลาโดยเฉพาะของ Render) แล้วให้เรียกยิงสเปค API สำเร็จรูปของเราไปที่เมธอด `POST /api/trigger-post` เพื่อความเสถียรสูงสุดและสั่งรันด่วนได้ทันที!
